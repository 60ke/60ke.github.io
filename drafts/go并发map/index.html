golang默认map不是并发安全的

(Why are map operations not defined to be atomic?)[https://golang.org/doc/faq#atomic_maps] After long discussion it was decided that the typical use of maps did not require safe access from multiple goroutines, and in those cases where it did, the map was probably part of some larger data structure or computation that was already synchronized. Therefore requiring that all map operations grab a mutex would slow down most programs and add safety to few. This was not an easy decision, however, since it means uncontrolled map access can crash the program.

The language does not preclude atomic map updates. When required, such as when hosting an untrusted program, the implementation could interlock map access.

Map access is unsafe only when updates are occurring. As long as all goroutines are only reading—looking up elements in the map, including iterating through it using a for range loop—and not changing the map by assigning to elements or doing deletions, it is safe for them to access the map concurrently without synchronization.

As an aid to correct map use, some implementations of the language contain a special check that automatically reports at run time when a map is modified unsafely by concurrent execution.

package main

import (
	"fmt"
	"sync"
)

func main() {
	var wg sync.WaitGroup
	type M struct {
		sync.RWMutex
		m map[int]int
	}
	tar := M{
		m: make(map[int]int),
	}
	for i := 0; i < 3; i++ {
		wg.Add(1)
		fmt.Println("maini:", i)
		go func() {

			defer wg.Done()

			tar.Lock()
			fmt.Println("goi:", i)
			tar.m[i] = i
			tar.Unlock()
		}()

	}
	var k int
	for k = 0; k < 1; k++ {
		fmt.Println("k:", k)
	}
	fmt.Println("maink:", k)
	wg.Wait()
	fmt.Printf("%v", tar.m)
}

观察代码,我们已经通过sync.RWMutex使结构体M支持读写锁。并在操作tar.m时通过tar.Lock()与tar.UnLock()来确保了数据的安全,但是还是出现了DATA RACE,同时我们可以看到tar.m的map数据并不是我们期望的[1:1,2:2,3:3]这是因为我们忽略了tar.m[i]中的i值来自main的goroutine,main的goroutine运行要比我们使用go关键字所创造的goroutine运行的快,我们的三次对map的赋值的key分别是 2,3,3所以我们的map只有2,3的key。我们可以通过传递i到goroutine中来解决这个错误:

go func(i int) {

	defer wg.Done()

	tar.Lock()
	fmt.Println("goi:", i)
	tar.m[i] = i
	tar.Unlock()
}(i)
